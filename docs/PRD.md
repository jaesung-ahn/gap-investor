# 부동산 투자 어플 PRD (초안)

## 1. 제품 개요

**제품명(가칭)**: 갭헌터 / 전세갭레이더  
**목적**: 전세가와 매매가의 갭 정보를 분석하여 임장 지역 및 매물 우선순위를 효율적으로 결정하도록 돕는 서비스  
**핵심 기능**
- 지역 기반 전세가/매매가 및 갭 정보 제공
- 갭 크기·전세가율·단순 수익률 등을 기준으로 매물/지역 랭킹 제공
- 임장 후보 리스트(북마크) 생성 및 관리
- (확장) 대출/자기자본을 고려한 수익성 시뮬레이션

---

## 2. 배경 및 문제 정의

### 2.1 투자자의 현재 문제
- 전세가/매매가 정보를 한 곳에서 비교하기 어려움
- 지역별 갭 규모 판단이 어렵고 감에 의존
- 여러 사이트/엑셀을 번갈아 사용해야 함
- 임장 후보지 선정 과정이 비효율적

### 2.2 해결하려는 문제
- “어디부터 임장을 가야 하는지”를 데이터 기반으로 결정할 수 있도록 정량화
- 갭·전세가율 등 핵심 지표를 자동 계산하고 시각화
- 임장 리스트를 구조적으로 관리할 수 있는 기능 제공

---

## 3. 타깃 사용자

1. 갭투자 입문~중급 개인 투자자  
2. 임장 기반 투자 의사결정 중심의 실전 투자자  
3. 여러 지역의 매물 지표를 빠르게 비교하려는 사용층

---

## 4. 핵심 가치 제안 (Value Proposition)

- **임장 NAVI**: 갭과 전세가율 기준으로 우선 가봐야 할 지역/단지를 자동 정렬  
- **직관적 데이터 도구**: 지도/랭킹/그래프 기반 판단  
- **의사결정 부담 축소**: 지표·리스크·변동 흐름을 요약 제공  

---

## 5. 주요 Use Case & User Journey

### UC1 – 지역 탐색
- 사용자가 “서울/수도권 중 갭 작은 곳”을 찾고 싶을 때  
- 지역 선택 → 갭/전세가율 정렬 → 우선순위 지역 확인

### UC2 – 특정 지역 분석
- 예: 안산 단원구 집중 분석  
- 해당 지역 단지별 평균 지표, 최근 시세 흐름, 갭 분포 확인

### UC3 – 임장 리스트 작성
- 탐색 → 필터링 → 북마크 → “임장 후보”로 정리

### UC4 – 수익성 시뮬레이션 (확장)
- 자기자본·대출 입력 → 특정 매물 예상 수익 계산

---

## 6. 범위 정의 (MVP vs 고도화)

### 6.1 MVP (1차 출시)

1. **지역 선택 및 탐색**
   - 시/구/동 단위 필터

2. **전세가/매매가/갭/전세가율 계산**
   - 단지 및 매물별 지표 표시

3. **정렬/필터 기능**
   - 정렬: 갭, 전세가율, 매매가  
   - 필터: 가격 범위, 전세가율 범위, 준공연차, 면적 구간

4. **단지/매물 상세 정보**
   - 기본 정보, 최근 1~3년 시세 그래프, 핵심 지표

5. **관심 리스트(북마크)**
   - 임장 후보 관리

6. **임장 체크리스트(정적 템플릿)**

---

### 6.2 향후 고도화(V2 이후)

1. **임장 플래너 (지도 기반 동선 추천)**  
2. **가격 변동·갭 변동 알림**  
3. **수익성 시뮬레이터(대출/자본 입력)**  
4. **리스크 인디케이터 (입주물량, 전세가율 과열 지역 등)**  
5. **메모/사진 업로드 기능**  

---

## 7. 기능 요구사항 (Functional Requirements)

### 7.1 데이터 및 지표 정의

**필드**
- 위치(시/구/동), 주소  
- 건물유형, 면적(전용), 준공년도  
- 매매가, 전세가  

**파생 지표**
- 갭 = 매매가 – 전세가  
- 전세가율 = 전세가 / 매매가 × 100  
- 단지 평균값 제공  
- 가격 변동률(고도화)

### 7.2 사용자 플로우

#### 지역 선택
1. 메인 → 지역 검색/선택  
2. 지역별 요약 지표 표시  
3. 단지/매물 리스트 이동

#### 리스트 탐색
1. 평균 지표(매매가·전세가율·갭) 노출  
2. 정렬/필터  
3. 북마크 가능

#### 상세 화면
1. 기본 정보  
2. 갭/전세가율  
3. 최근 시세 그래프  
4. 메모/북마크 기능

#### 임장 리스트
1. 북마크 매물 모음  
2. 정렬(갭/전세가율/지역)  
3. 주소 복사·지도앱 연동  

---

### 7.3 화면 요구사항 요약

**메인 화면**
- 지역 검색  
- 갭 작은 인기 지역 TOP 5  
- 최근 본 매물

**지역 리스트**
- 평균 지표  
- 정렬/필터  
- 단지 리스트

**상세 화면**
- 가격 지표  
- 그래프  
- 단지 특성  
- 관심 추가

**임장 후보 화면**
- 북마크 리스트  
- 메모 기능(고도화)

---

## 8. 비기능 요구사항 (Non-Functional Requirements)

- 성능: 주요 화면 로딩 3초 이내  
- 가용성: 99% 이상  
- 보안: 계정 기반 데이터 분리  
- 확장성: 데이터 소스 교체 가능 구조  
- 호환성: 모바일 앱 우선, 추후 웹 확장  

---

## 9. 성공 지표 (KPI)

**단기 (3~6개월)**  
- MAU  
- 세션당 평균 조회 단지 수  
- 임장 후보 리스트 생성 비율  

**중기 (6~12개월)**  
- 7일/30일 재방문율  
- 알림 기능 사용률  
- 사용자 만족도(NPS)  

---

## 10. 리스크 및 고려사항

1. 데이터 품질 문제 (시세 최신성·정확성)  
2. 부동산 정책 변화에 따른 지표 해석 위험  
3. 투자 추천으로 오해될 가능성 → 명확한 안내 필요  
4. 기능 과부하로 초보자 혼란 가능 → MVP 최소화 전략 필요  

---
